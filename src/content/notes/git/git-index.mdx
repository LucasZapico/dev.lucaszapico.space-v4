
### Revert a Committed Change with reset HEAD

[How to Git Reset to HEAD - devconnected](https://devconnected.com/how-to-git-reset-to-head)




Using the `--hard` flag will remove all changes in the working directory and index. While using the `--soft` flag will keep the changes in the working directory and index.

#### Go Back Commits Without loosing Changes 

```sh
git reset --soft HEAD~<number of commits> 
```

### Go Back Commits and Remove Changes 

#### Discard all current changes 

```sh
git reset --hard HEAD
```

#### Discard all current changes and go back to a specific commit

Go back to single commit before `HEAD`

```sh
git reset --hard HEAD^

```

or 

```sh
git reset --hard HEAD~1 
```

Go back two commits before `HEAD`

```sh
git reset --hard HEAD~2
```

```sh 
git reset HEAD~<number of commits> --hard 
```

### Delete Local and Remote Branches

```shell
// delete branch locally
git branch -d localBranchName

// delete branch remotely
git push origin --delete remoteBranchName
```

## Stash 

### Stash Changes 

```sh
git stash 
```

### Stash Changes with Message

```sh
git stash save "message"
```

### List Stashes

```sh
git stash list
```

### Adding Stash Change 

`apply`
: `apply` adds the stash changes to the working directory but keeps the stash in the stash list.

`pop` 
: `pop` adds the stash changes to the working directory and removes the stash from the stash list.

#### Apply stash

```sh 
git stash apply stash@{<stash number>}
```

```sh 
git stash apply stash@{1}
```

### Pop Stash


```sh 
git stash pop stash@{<stash number>}
```

```sh 
git stash pop stash@{1}
```

### Drop Stash

`drop` removes a stash from the stash list completely.

```sh
git stash drop stash@{<stash number>}
```

```sh
git stash drop stash@{1}
```

### Clear Stash

> warning should be use with caution as it removes all stashes

`clear` removes all stashes from the stash list.

```sh
git stash clear
```
